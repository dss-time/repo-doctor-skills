# 示例

- `test_first`：“实现账户锁定前，先补一个证明第六次失败登录被拒绝的最小测试。”必须观察到真实预期行为失败，生产实现另行转交。
- `regression_after_fix`：“空会话修复已经验证，现在补回归测试。”在修复后状态运行并报告安全敏感性证据，不得编造修复前红灯。
- `characterization`：“替换解析器前，用测试固定当前 CSV 转义行为。”初次通过有效；准确说明冻结了什么行为。
- 阻塞场景：答案会改变模式时，询问行为是尚未实现、已经修复，还是需要保留的旧行为。
- 反例：“修复生产代码中的空指针。”应使用 `safe-fix-implementation`。
