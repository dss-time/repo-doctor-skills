# Examples

- “Before writing a spec, clarify whether deletion is reversible, who may restore records, and what compatibility is required.”
- “Read the existing API and glossary first. Ask only decisions the repository does not answer.”
- A clear request such as “Change this typo in one error message; no API behavior changes” should receive a short confirmation, not a broad questionnaire.
- Non-trigger: “The decisions are closed; write the implementation-ready specification.” Use `requirements-to-spec`.
